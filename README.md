#快递管理系统（Express System）
**
项目简介
一款基于 C++ 面向对象开发、SQL 数据库支撑的轻量化快递管理系统，采用「抽象基类 + 多态 + 三层架构」设计，核心实现快递包裹从寄件到签收的全生命周期管理，支持管理员、快递员、普通用户三类角色协同操作。
核心设计原则
严格职责分离：
界面层（UI）：仅负责菜单显示、用户输入、结果打印，不包含任何业务逻辑
业务层（Service）：仅负责权限校验、核心流程实现，不包含任何界面相关代码（无 cout/cin）
数据层（DAO）：仅负责数据库连接、SQL 执行、数据读写，与界面 / 业务解耦
解耦可扩展：通过抽象基类统一接口，多态实现角色逻辑隔离，支持界面灵活替换（如 CLI 换 GUI）
数据一致性：所有角色共享同一数据库连接，账户信息单表存储，避免数据冗余
易用性：命令行界面（CLI）轻量化交互，操作流程清晰
