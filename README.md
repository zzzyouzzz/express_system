# 快递管理系统（Express System）




## 项目简介

一款基于 C++ 面向对象开发、SQL 数据库支撑的轻量化快递管理系统，采用「抽象基类 + 多态 + 三层架构」设计，核心实现快递包裹从寄件到签收的全生命周期管理，支持管理员、快递员、普通用户三类角色协同操作。

### 核心特性



* 📋 **严格职责分离**：界面层仅负责显示输入，业务层专注逻辑实现，数据层统一数据操作

* 🔧 **高可扩展性**：抽象基类定义统一接口，支持角色扩展与界面替换（CLI/GUI 无缝切换）

* 📊 **数据一致性**：所有角色共享数据库连接，账户信息单表存储，避免数据冗余

* 🎯 **轻量化交互**：命令行界面简洁直观，操作流程清晰，无额外依赖

## 技术栈



| 模块   | 技术选型                           | 说明                        |
| ---- | ------------------------------ | ------------------------- |
| 开发语言 | C++（C++11+）                    | 面向对象特性（抽象类、多态、继承）支撑核心架构   |
| 数据库  | SQL（MySQL 5.7+/SQLite 3.0+ 兼容） | 关系型数据库，结构化存储账户与包裹数据       |
| 界面交互 | 命令行界面（CLI）                     | 封装为独立类，仅负责「显示 + 输入」，无业务逻辑 |
| 架构模式 | 三层架构（界面层 / 业务层 / 数据层）+ OOP     | 抽象基类定义接口，子类实现具体业务逻辑       |
| 数据交互 | 数据库引用注入 + 实体对象传递               | 层间通过实体对象传递数据，禁止零散参数       |

## 系统架构

### 整体架构



```
界面层（MenuUI 封装类）→ 业务层（Account 抽象基类+角色子类）→ 数据层（Database 类+SQL 数据库）
```

### 分层职责明细



| 分层      | 核心职责                                                                                  | 绝对禁止操作                                                | 核心实现                                                                       |
| ------- | ------------------------------------------------------------------------------------- | ----------------------------------------------------- | -------------------------------------------------------------------------- |
| **界面层** | 1. 所有菜单显示（登录 / 角色 / 操作子菜单）2. 用户输入获取与合法性校验 3. 业务结果打印（成功 / 失败 / 数据展示）. 调用业务层接口触发逻辑      | 1. 禁止处理业务逻辑（权限判断 / 状态校验）2. 禁止直接操作数据库 3. 禁止在其他层写界面代码   | 封装为 `MenuUI` 类： 持有数据库引用（注入业务层） 持有账户基类指针（多态调用）- 菜单通过独立方法实现                  |
| **业务层** | 1. 角色权限校验（基于 `role` 字段）. 核心业务逻辑实现（寄件 / 揽收 / 签收等）>3. 调用数据层读写数据。返回标准化结果（成功 / 失败 / 实体对象） | 1. 禁止包含界面相关代码 2. 禁止直接执行 SQL 语句。禁止存储界面相关状态             | 抽象基类 `Account` 定义统一接口，子类实现角色专属逻辑：Admin`：包裹管理、账户管理rier`：包裹揽收、查询 \`：寄件、查件、签收 |
| **数据层** | 1. 数据库连接初始化与管理（全局单实例）. SQL 执行（增删改查）>3. 结果封装为实体对象返回 4. 数据库异常处理                         | 1. 禁止处理业务逻辑（权限判断 / 状态流转）>2. 禁止包含界面相关代码。禁止返回原始 SQL 结果集 | `Database` 类封装所有数据库操作，统一提供 CRUD 接口，所有角色共享连接                                |

### 类体系结构



```
├── Database（数据层：数据库操作封装）

├── Account（业务层：抽象基类，定义统一接口）

│   ├── Admin（管理员子类：实现管理员业务）

│   ├── Courier（快递员子类：实现快递员业务）

│   └── User（普通用户子类：实现用户业务）

├── Entity（实体类：仅存数据，无逻辑）

│   ├── User（用户实体：存储账户信息）

│   ├── Express（快递实体：存储包裹信息）

│   └── Item（物品实体：普通物品/易碎品/图书等）

└── MenuUI（界面层：仅负责显示与输入）
```

| ------------ | ------------- | --------- | ----------------------------- |
| user\_id     | INT           | 主键、自增     | 唯一账户 ID                       |
| username     | VARCHAR(50)   | 非空、唯一     | 登录用户名                         |
| password     | VARCHAR(50)   | 非空        | 登录密码（建议用 MD5 加密存储）            |

| create\_time | DATETIME      | 非空、默认当前时间 | 注册时间                          |

| receiver\_phone   | VARCHAR(20)  | 非空        | 接收人手机号                               |
| receiver\_address | VARCHAR(200) | 非空        | 接收人详细地址                              |

## 交互流程

### 示例 1：快递员包裹揽收流程



1. 界面层：显示快递员菜单及「包裹揽收」选项

2. 界面层：获取用户输入的包裹编号，做基础合法性校验（非空等）

3. 界面层：调用业务层查询接口，获取包裹信息

4. 业务层：调用数据层查询包裹，返回快递实体对象

5. 界面层：校验包裹是否存在，若不存在则打印提示

6. 界面层：调用业务层揽收接口，传递包裹实体

7. 业务层：校验包裹状态（仅待揽收可操作），更新状态为「已揽收」

8. 业务层：调用数据层执行更新操作，返回操作结果

9. 界面层：打印揽收成功 / 失败信息

### 示例 2：普通用户寄件流程



1. 界面层：显示用户菜单及「发送快递」选项

2. 界面层：获取用户输入的物品信息、接收人信息，做基础合法性校验

3. 界面层：封装为快递实体对象（仅赋值，无业务逻辑）

4. 界面层：调用业务层寄件接口，传递快递实体

5. 业务层：生成唯一包裹编号、补全寄件时间

6. 业务层：调用数据层插入包裹信息，返回操作结果与包裹编号

7. 界面层：打印寄件成功 / 失败信息及包裹编号

## 快速开始

### 环境要求



* 编译器：GCC 4.8+/Clang 3.3+/MSVC 2015+（支持 C++11）

* 数据库：MySQL 5.7+/SQLite 3.0+

* 依赖：MySQL 环境需链接 `libmysqlclient` 库，SQLite 环境需链接 `libsqlite3` 库

### 编译运行步骤



1. **获取源码**



```
git clone <项目仓库地址>

cd express\_system
```



1. **初始化数据库**

* MySQL 环境：



```
mysql -u root -p .sql
```



* SQLite 环境：



```
sqlite3 express.db init.sql
```



1. **配置数据库连接**

* 修改数据层配置文件（如 `Database.h`）中的数据库地址、用户名、密码（MySQL 环境）

1. **编译源码**



```
\# MySQL 环境

g++ src/\*.cpp -o express\_system -lmysqlclient -std=c++11

\# SQLite 环境

g++ src/\*.cpp -o express\_system -lsqlite3 -std=c++11
```



1. **运行程序**



```
\# Linux/Mac

./express\_system

\# Windows（CMD）

express\_system.exe
```

## 开发规范

### 界面层规范



* 所有菜单必须封装在 `MenuUI` 类中，通过独立方法实现

* 仅在界面层使用输入输出相关操作，其他类禁止出现界面相关代码

* 输入需做合法性校验（如非空、手机号格式等），但不做业务逻辑校验

### 业务层规范



* 仅实现业务逻辑，返回标准化结果（成功 / 失败 / 实体对象）

* 禁止直接操作数据库，必须通过数据层接口

* 禁止包含任何界面相关代码

### 数据层规范



* 数据库操作统一封装，全局单实例，避免重复连接

* 仅返回实体对象，禁止返回原始 SQL 结果

* 负责数据库异常处理（如连接失败、SQL 执行错误）

### 层间交互规范



* 仅允许「界面层→业务层→数据层」的单向调用，禁止反向调用

* 数据传递仅使用实体对象，禁止传递零散参数、SQL 语句
